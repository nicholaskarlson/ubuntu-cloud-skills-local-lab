# Learning Contract

This repo follows four rules.

## 1. Local first

The first lessons should run on a local machine.

Students should not need a paid cloud account to begin.

## 2. Free-account friendly

The core path should avoid paid Docker features.

Docker Pro may be useful for the author, but it should not be required for students.

## 3. Provider portable

The concepts should transfer to many places:

- a local Ubuntu machine
- Windows with WSL2 Ubuntu
- DigitalOcean or similar VPS providers
- AWS Lightsail or EC2
- other Linux servers

## 4. Proof first

Every meaningful workflow should produce evidence:

- version receipts
- command receipts
- test receipts
- cleanup receipts

Receipts should never include passwords, tokens, SSH private keys, or other secrets.
